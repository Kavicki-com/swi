## Task 1: AppLayout — replace composed header with DS Header

The current `AppLayout.tsx` composes the admin header inline (Logo + Text(name) + Button(Sair)). Replace with DS `Header` + `HeaderUserInfo` (the vitals widget that S1 erroneously rejected).

### Required props for HeaderUserInfo

From the Figma:
- `bpm: number` — heart rate of the logged-in admin (e.g., 98)
- `pressure: string` — blood pressure (e.g., "12/8")
- `avatarUri?: string` — admin avatar
- `name?: string` — admin name (Bryan in the Figma)

The current `User` type doesn't have bpm/pressure. For S1.7 add them as optional fields with mocked defaults (S2 will add real biometric data sourcing). Update:
- `services/types/index.ts` — add `bpm?: number`, `pressure?: string` to User
- `services/mockApi/seed.ts` — give SEED_ADMIN realistic mock vitals (e.g., bpm: 78, pressure: "12/8")
- `useAuth.tsx` — no change (passes through)

Update `AppLayout.tsx` to use DS Header:

```tsx
<Header
  logoSize="m"
  logoType="complete"
  bpm={user?.bpm ?? 78}
  pressure={user?.pressure ?? "12/8"}
  avatarUri={user?.avatarUri}
/>
```

Where does the **Sign-out** button go? Figma shows none in the header. Move sign-out to the SideMenu footer (or add a context menu to the avatar). Per Figma, the SideMenu has a "Pesquisar Contatos" search + chat list + user list at the bottom — sign-out probably belongs in a dropdown from the avatar. Options:
- (a) Avatar click → dropdown with "Sair" — needs a Menu component (check DS; if absent, defer)
- (b) Add a small "Sair" GhostButton in SideMenu footer — pragmatic for S1.7 fidelity
- (c) Remove sign-out from header entirely; rely on `/logout` route (less discoverable)

Pick (b) for S1.7. Decide with design at S2.

### Tests update

`AppLayout.test.tsx` currently checks:
- "renders outlet content for authenticated user"
- "shows the user full name in the header"
- "renders all 9 navigation entries"
- "signs out and navigates to /login"

Test 2 may need adjustment — if HeaderUserInfo doesn't render `full_name` directly (it's a vitals widget; `name` is shown next to avatar). Check Figma + DS:
- If Figma shows the name "Bryan" in the header → keep test, lookup updated DS prop name
- If Figma doesn't show the name → drop assertion or move to AppLayout footer where Sair lives

Test 4 lookup for sign-out button moves from header to wherever we placed it.

### Acceptance

- Sign-out works from new location
- Visual matches Figma header strip

## Task 3: Dashboard KPI row (rewrite)

Current: 4 BigNumbersCards (total, em alerta, alertas abertos, críticos) + 4 status cards (good/alert/low/offline). Total 8 cards stacked.

Figma: ~5-6 cards in a horizontal row:
1. **Funcionários** — composite with sub-numbers + status icons (good/alert/low badges)
2. **Sinais vitais** — single big number (e.g., 512)
3. **Taxa de desgaste** — single big number (e.g., 512) with green accent
4. **Alertas urgentes** — count + "Necessita atenção" sublabel
5. **Alertas comuns** — count
6. **Pacientes em sigilo** — count

Some are composite cards (multiple numbers). DS `BigNumbersCard` accepts a single value+label. May need:
- DS `BigNumbersCard` with a new `subValues` prop (DS bump) OR
- Compose Card with multiple BigNumbersCards inside OR
- New DS component `KpiCard` with composite layout

Decide based on audit. For now, pragmatic: render 6 BigNumbersCards in a flex-row+wrap. Reserve composite layout (Funcionários card with status sub-icons) for DS bump.

`dashboard.summary` mockApi may need to expose more aggregate values (sinais vitais count, taxa de desgaste, etc.). Update the type + mock seeds.

### Acceptance

- KPI row matches Figma horizontally (5-6 cards visible without wrap on 1366px width)
- All values come from `dashboardApi.summary` (no hardcoded numbers)
- Tests assert the new fields

1. **DS bumps required** — Map preview, composite KPI card, ChipGroup may all need DS additions. Each adds time. Build a v0.1.4 plan.
