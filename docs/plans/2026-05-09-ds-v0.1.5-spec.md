# `@kavicki/swi-design-system` v0.1.5 — Header fidelity bump

**Date:** 2026-05-09
**Source repo:** `github:Kavicki-com/swi-design-system`
**Consuming app:** `swi-admin` (worktree `Documents\SWI`)
**Trigger:** S1.7 dashboard fidelity loop revealed gaps in DS `HeaderUserInfo` that previously forced a local override (`AppHeaderUserInfo.tsx`). That override has been deleted per project rule. v0.1.5 closes the gaps so the consumer can return to using `HeaderUserInfo` directly.

---

## Goal

Make `HeaderUserInfo` capable of matching Figma frame `4:2` (fileKey `bzDUuPdSiKgl5xucBH0IYE`) without any local override in consuming apps:

- Render the **Figma-exported heart and pressure SVG icons** instead of the current Material Symbols defaults.
- Allow consumers to **disable or recolor the avatar border** (currently hardcoded `<Avatar bordered />` with default `theme.content.dark`).

All changes are **additive and backward-compatible** — every new prop has a default that preserves the v0.1.4 visual exactly. No existing consumer breaks.

### 3. `src/components/HeaderUserInfo/HeaderUserInfo.types.ts`

Add four optional props:

```ts
import type { IconName } from '../../icons/paths';

export interface HeaderUserInfoProps {
  // ...existing
  bpm: number;
  pressure: string;
  progress?: number;
  avatarUri?: string;
  bpmUnit?: string;
  accessibilityLabel?: string;
  testID?: string;

  // NEW in v0.1.5 — all optional, all backward-compatible
  /** Icon name for the BPM stat. Default: `'favorite'` (Material). */
  heartIconName?: IconName;
  /** Icon name for the pressure stat. Default: `'monitor_heart'` (Material). */
  pressureIconName?: IconName;
  /** Whether the avatar has a visible ring border. Default: `true` (current). */
  bordered?: boolean;
  /** Avatar border color. Default: `theme.content.dark` (current). */
  borderColor?: string;
}
```

### 4. `src/components/HeaderUserInfo/HeaderUserInfo.tsx`

Wire the new props through. Defaults preserve v0.1.4 behavior:

```tsx
export const HeaderUserInfo = forwardRef<View, HeaderUserInfoProps>(
  (
    {
      bpm,
      pressure,

      heartIconName = 'favorite',

            </StatItem>
```

## Open questions for the DS owner

1. Should the new icon names follow Material's convention (`heart_filled` / `vitals_pulse`) or the brand's own naming? My placeholder names assume the former.
2. The `vitals_pulse` SVG uses a `<g clip-path>` masking two paths to a 20x20 inner region. Two options:
   - (a) Pre-bake the clip into the path data using a tool (e.g. `svgo` with the right plugin) and register a single clipped path.
   - (b) Extend the DS Icon primitive to support multiple paths or a clip-path attribute.
   Recommend (a) — minimal change to the primitive and matches the existing single-path contract.
3. Should `bordered` and `borderColor` defaults stay at `true` / `theme.content.dark`, or is this the moment to revisit the default (e.g. ship as `false` / `transparent` since Figma frame 4:2 doesn't show a visible border)? Backward compat says keep current defaults; aesthetic argues for changing. The user's instinct this session was that the border shouldn't visually exist — but other consumers may differ.
