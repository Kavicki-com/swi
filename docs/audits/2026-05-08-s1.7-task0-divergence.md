## Gaps and decisions

### Gap A — `BigNumbersCard` lacks sublabel + tone

**Figma:** "Alertas urgentes" card has a small "Necessita atenção" pill/sublabel underneath the count. "Taxa de desgaste" uses a green accent on the icon area.
**DS today:** `BigNumbersCardProps = { value, label, icon?, iconColor?, fullWidth?, ... }`. No `sublabel` and no theming variant.

**Decision for S1.7:** Compose locally — wrap `BigNumbersCard` in a `View` and overlay a small `<Text>` row for the sublabel; pass `iconColor` for accent. Pixel result will match the Figma at ~95%. Defer the proper DS prop to a future v0.1.4 (`sublabel?: string`, `tone?: 'default'|'warning'|'success'`). **No DS bump in this sprint.**

### Gap B — `EmployeeOverviewCard` lacks `⋮` action menu

**Figma:** Right-column employee rows include a 3-dot kebab menu on the right (likely opens "perfil / silenciar / etc.").
**DS today:** `onLocationPress` and `onPress` only; no `onMorePress` and no kebab affordance.

**Decision for S1.7:** Render rows without the kebab. Document as deferred. Defer the new prop to v0.1.4. **No DS bump in this sprint.**

### Gap C — KPI "Funcionários" composite card

**Figma:** The leftmost KPI is a composite — primary count "3 Funcionários", three small avatars, secondary count "4 Funcionários", with status badges around. It is not a single big-number card.
**DS today:** `WorkersInfoCard` exists but is for individual employee detail, not for status aggregation.

**Decision for S1.7:** Compose locally with `View` + nested `BigNumbersCard` and `AvatarGroup`. **No DS bump in this sprint.** (A future `KpiCompositeCard` would belong in DS v0.2 once a second screen needs the same shape.)

### Gap D — Map preview banner

**Figma:** Top of dashboard main column: an aerial image with a center marker (no visible "Acessar mapa" text in this snapshot, but plan describes a CTA overlay).
**DS today:** `MapControl` is the wrong primitive (overlay filter panel).

**Decision for S1.7:** Use DS `Image` + DS `Button` (overlay positioned absolute) for the banner. Click → `navigate('/maps/general')`. Image source can be a static asset for now (`assets/dashboard-map-preview.png`) until S2 wires real Mapbox/Leaflet. **No DS bump in this sprint.**

### Gap E — Sign-out location

**Figma:** Header has no "Sair" button. SideMenu in Figma shows chat list + "Esperando chat" but no sign-out either.
**DS today:** `SideMenu` accepts items only; no built-in footer slot.

**Decision for S1.7:** Per plan option (b), add a small DS `Button` (variant=ghost, label="Sair") rendered just **below** `SideMenu` inside `AppLayout`'s sidebar `View`. Pragmatic and discoverable.
